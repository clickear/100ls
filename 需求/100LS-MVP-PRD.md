## 1. 背景与目标

### 1.1 背景

100LS（Language Shadowing）是张同完验证有效的语言学习方法，核心逻辑是：选择一部喜爱的影片，反复观看同一内容100遍，在听、读、说的循环中自然建立语感。该方法已被大量学习者验证有效，但实操中存在三个核心痛点：

1. **工具门槛高**：学习者需要手动用播放器分段、反复拖拽进度条、切换字幕文件，操作繁琐且容易打断学习心流。
2. **缺少阶段化指引**：100遍只是数字，学习者不清楚每一遍该做什么（看中英字幕？纯英文字幕？盲听？），导致机械重复、效率低下。
3. **坚持成本高**：独自学习缺少反馈和进度感，大部分人完成不到10遍就放弃，远未达到100遍的临界点。

目前市面上没有专门为100LS方法设计的工具。现有英语学习App以碎片化课程为主，无法满足"同一内容深度重复100遍"这一特殊场景。因此，需要一款垂直工具来降低100LS的学习门槛。

### 1.2 目标用户

| 用户角色 | 特征描述 | 核心诉求 | 使用场景 |
|---------|---------|---------|---------|
| 主角色：有一定基础的自学者 | 词汇量约2000-5000，能看懂简单字幕，但听力/口语有瓶颈，已了解100LS方法论但尚未完整实践过 | 用趁手的工具降低100LS实操门槛，获得清晰的学习路径和进度反馈，最终能坚持完成100遍 | 每天固定30-60分钟沉浸式学习，使用自己已有的影片资源（如电影、剧集、TED演讲） |
| 次要角色：多语言爱好者 | 不局限于英语，想将100LS方法应用于日语、韩语、法语等 | 工具支持多语言字幕和多语种视频 | 学习非英语内容时，同样使用同一套100LS流程 |

### 1.3 业务目标与成功指标

> 本产品为自用工具，核心目标为验证100LS垂直工具的产品价值，无商业化指标。

| 目标 | 衡量指标 | 目标值 | 监测方式 |
|------|---------|--------|---------|
| 降低工具门槛 | 用户完成首次学习所需的准备时间 | < 3分钟（从打开App到开始第一遍学习） | 本地埋点统计 |
| 提升100遍完成率 | 单部影片完成100遍的用户比例 | > 30% | 本地进度数据 |
| 减少学习中断 | 用户连续中断学习的天数 | 连续7天未学习时触发提醒 | 本地日历埋点 |
| 提升学习效率 | 单阶段（10遍）的平均完成时长 | < 5天/阶段 | 本地进度数据 |

## 2. 需求概述

打造一款支持粘贴视频URL自动下载、自动解析中英双语字幕、按10阶段自动推进的100LS学习工具，帮助用户零门槛开始并完成100遍深度学习。

### 2.1 MVP范围界定

**本期必须包含（P0）**：
- **在线视频URL下载（支持YouTube、Bilibili等主流平台）+ 自动解析中英双语字幕**
- 本地视频导入 + 字幕加载/自动生成
- 100LS专用播放器（AB循环、变速、字幕切换、逐句播放）
- 阶段化学习引擎（10个阶段自动推进，每阶段解锁不同字幕模式和任务）
- 进度追踪（100遍计数、学习时长、完成状态可视化）
- **跨平台支持：桌面端（macOS/Windows）+ 移动端（iOS/Android）**

**本期强烈建议包含（P1）**：
- 习惯养成系统（连胜机制、每日提醒、基础徽章）
- 词汇辅助（点击字幕查词、生词标记）
- 跨设备数据导出/导入（手动迁移学习记录）

**本期不做（明确排除）**：
- Chrome插件形态
- 内置版权影视库（依赖外部版权授权）
- 社区/社交功能（跟读PK、UGC内容）
- AI语音评测与发音打分
- 学习成果视频生成
- 班级/教师管理后台
- 自动云端同步（数据本地存储，跨设备通过导出/导入手动迁移）
- **视频自动切分（长视频按场景/章节切分为多个短视频片段）—— 纳入 V2.0 后续迭代**

## 3. 功能详细设计

### 3.1 视频导入与字幕管理

**功能描述**：允许用户通过粘贴视频URL自动下载视频和字幕，或导入本地视频文件和字幕文件，建立学习内容库。字幕系统支持中英双语，为渐进式字幕展示提供数据基础。

**用户故事**：
- 作为自学者，我希望粘贴一个YouTube/B站链接就能自动下载视频和双语字幕，以便快速开始100LS学习。
- 作为自学者，我希望导入自己电脑里的电影文件，以便用喜爱的内容练习100LS。
- 作为自学者，我希望App能自动为视频生成中英双语字幕，以便我能在不同阶段使用中英文对照学习。
- 作为多语言学习者，我希望导入不同语言的视频和字幕，以便用100LS学习多种语言。

**优先级**：P0

**业务规则**：
1. **导入来源双通道**：
   - **URL下载**：用户粘贴视频页面URL（支持YouTube、Bilibili等主流平台），App自动解析并下载视频文件和可用字幕。
   - **本地导入**：用户从本地文件系统选择视频文件和字幕文件。
2. 支持导入的视频格式至少包含 MP4、MKV、MOV、AVI。
3. 支持导入的字幕格式至少包含 SRT、VTT、ASS。
4. **中英双语字幕获取策略**（按优先级依次尝试）：
   - **策略A（最优）**：若平台提供中英双语字幕轨道（如B站CC字幕的双语选项、YouTube的多语言字幕），自动下载英字和中字两个字幕文件，按时间轴自动对齐合并为一条双语字幕（英文上/中文下，或根据用户设置调整）。
   - **策略B（次优）**：若平台仅提供英文字幕，自动下载英文字幕，同时下载视频音频，通过语音识别生成英文字幕的时间轴，然后用机器翻译将英文字幕翻译为中文字幕，最终合并为双语字幕。
   - **策略C（保底）**：若平台无任何字幕，下载视频音频后，通过语音识别生成英文字幕，再翻译为中文字幕，合并为双语字幕。
   - **策略D（本地补充）**：用户导入本地视频时，App自动检测同目录下是否存在同名字幕文件（如 video.mp4 对应 video_en.srt / video_cn.srt），若存在中英两个字幕文件则自动对齐合并；若仅有一个字幕文件，则按策略B/C补充缺失语言。
5. **双语字幕合并规则**：
   - 以英文字幕的时间轴为基准，中文字幕按最近时间戳对齐到对应的英文句子上。
   - 若中英文句子数量不一致（如中文翻译将两句英文合并为一句），以英文句子为单位拆分，中文重复显示或标记为"[多句合并]"，允许用户在时间轴编辑器中手动调整对应关系。
   - 合并后的双语字幕格式：每句包含英文原文、中文译文、开始时间、结束时间。
6. 自动生成的字幕允许用户逐句微调时间轴（±500ms）和文本内容。
7. 字幕文件仅保存在用户本地，不上传至任何服务器。
8. 每个视频项目需记录：影片名称、来源URL（如有）、语言、总时长、导入时间、关联字幕文件路径、字幕语言组合。
9. 视频项目支持删除，删除时询问是否同时删除关联的本地视频和字幕文件（默认不删除视频，仅删除字幕）。
10. **URL下载限制**：单视频时长上限为2小时（超过时提示"该视频较长，下载和字幕生成可能需要较长时间，是否继续？"）；同时下载队列最多3个，超出时需等待。

**交互流程**：

**路径A：URL下载（主要来源）**
1. 用户从首页点击"粘贴链接下载"，进入URL输入页。
2. 用户粘贴视频页面URL（如 YouTube、Bilibili 链接），点击"解析"。
3. App解析URL，展示视频元数据（标题、时长、封面、平台来源），用户确认后点击"开始下载"。
4. App同时启动视频下载和字幕抓取：
   - 展示双进度条："视频下载中（X%）"和"字幕解析中"。
   - 若平台提供中英双语字幕：直接下载并自动对齐合并。
   - 若平台仅提供英文字幕：下载英文字幕后，自动翻译生成中文字幕并合并。
   - 若平台无字幕：下载完成后提取音频，通过语音识别生成英文字幕，再翻译为中文字幕并合并。
5. 下载和字幕处理完成后，进入预览确认页，展示视频前30秒的双语字幕效果（中英对照显示）。
6. 用户可点击"确认"完成导入，或点击"调整字幕"进入时间轴编辑器。
7. 完成导入后，视频项目出现在首页列表，状态为"未开始"。

**路径B：本地导入**
1. 用户从首页"开始学习"或"+ 导入视频"入口进入导入流程。
2. 系统弹出文件选择器，用户选择本地视频文件：
   - **桌面端**：系统文件选择器（Finder/文件资源管理器），直接选择本地视频文件。
   - **移动端**：调用系统文件选择器（iOS Files App / Android 系统文件管理器），或从相册/下载目录中选择视频文件。支持"分享到App"方式导入（用户在其他App中点击分享，选择本App导入）。
3. 系统检测同目录字幕文件（桌面端）或检测文件名匹配的字幕文件（移动端从同一目录）：
   - 若检测到中英双语字幕文件：自动加载、对齐合并并进入预览确认页。
   - 若检测到单语字幕文件：加载该字幕，提示"是否自动生成缺失语言的翻译字幕？"
   - 若未检测到字幕：提示"未找到字幕，是否自动生成中英双语字幕？"
4. 若选择生成字幕，系统显示进度条（"正在识别语音并翻译，预计需要X分钟"），完成后进入预览确认页。
   - **移动端特别提示**：由于设备性能限制，自动字幕生成建议在Wi-Fi环境下进行，且处理时长可能为桌面端的1.5-2倍。超过15分钟的视频建议在桌面端完成字幕生成后再通过导出/导入迁移到移动端。
5. 预览确认页展示视频前30秒的字幕效果（中英双语对照），用户可点击"确认"完成导入，或点击"调整字幕"进入时间轴编辑器。
6. 完成导入后，视频项目出现在首页列表，状态为"未开始"。

**异常处理**：

| 异常场景 | 处理方式 | 用户提示 |
|---------|---------|---------|
| 导入的视频文件无法解码 | 提示用户更换视频格式或重新编码 | "该视频格式暂不支持，请转换为MP4后重试" |
| 自动字幕生成失败（音频质量差/无语音） | 允许用户手动上传字幕文件，或标记为"无字幕"状态 | "语音识别失败，建议手动添加字幕文件" |
| 字幕时间轴严重偏移 | 在时间轴编辑器中提供"整体偏移"滑块，一键对齐 | "字幕与画面不同步？使用整体偏移快速调整" |
| 导入的视频文件超过系统存储空间 | 计算并提示所需空间 | "剩余空间不足，需要至少X GB，请清理后重试" |
| **URL解析失败（链接无效或平台不支持）** | 提示用户检查链接，或改用本地导入方式 | "无法解析该链接，请检查URL是否正确，或尝试下载到本地后导入" |
| **URL下载过程中网络中断** | 自动恢复下载（支持断点续传），失败3次后提示用户重试 | "下载中断，正在尝试恢复...（X/3次）" |
| **平台无可用字幕且语音识别失败** | 允许用户先无字幕播放，或手动上传字幕文件 | "该视频暂无可用的自动字幕，语音识别也未成功。你可以先无字幕学习，或手动添加字幕" |
| **双语字幕对齐后中英文句数严重不匹配（差异>30%）** | 标记为"字幕对齐异常"，在时间轴编辑器中高亮差异区域，提示用户手动校对 | "中英字幕句数差异较大，已标记差异区域，建议手动调整对应关系" |
| 字幕时间轴严重偏移 | 在时间轴编辑器中提供"整体偏移"滑块，一键对齐 | "字幕与画面不同步？使用整体偏移快速调整" |
| **移动端：系统限制无法访问某些目录** | 引导用户先将视频复制到Files/下载目录，或改用"分享到App"方式导入 | "无法访问该位置，请先将视频保存到文件管理器中再导入" |
| **移动端：iOS相册视频无法直接读取** | 通过系统PhotoKit/Storage Access Framework获取视频副本的临时访问路径 | "正在从相册导入，请稍候" |
| 用户选择的字幕文件编码格式异常 | 自动尝试UTF-8/GBK/Big5解码，失败则提示手动转换 | "字幕编码无法识别，请保存为UTF-8格式后重试" |

### 3.2 100LS 专用播放器

**功能描述**：为100LS方法定制的视频播放器，核心能力包括AB循环、变速播放、多字幕模式切换和逐句播放。

**用户故事**：
- 作为自学者，我希望设置AB循环反复观看同一段台词，以便熟悉语音语调和句子结构。
- 作为自学者，我希望在不同阶段切换字幕类型，以便逐步从依赖字幕过渡到盲听。
- 作为自学者，我希望降低播放速度跟读，以便跟上母语者的语速。

**优先级**：P0

**业务规则**：
1. **AB循环**：用户可标记起点A和终点B，播放器在A-B区间内自动循环播放。AB区间最短不少于1秒，最长不超过5分钟。支持清除AB标记。
2. **变速播放**：支持0.5x、0.75x、1.0x、1.25x、1.5x五档变速。变速时保持音调不变（变速不变调）。
3. **字幕切换（基于中英双语字幕数据）**：播放器读取影片的双语字幕数据，支持以下五种显示模式：
   - **中英双语字幕**：英文原文在上，中文译文在下（或根据用户设置互换位置），方便初期理解剧情和句意对应关系。
   - **纯英文字幕**：仅显示英文原文，隐藏中文译文，用于训练英文阅读能力和脱离中文依赖。
   - **关键词提示字幕**：仅保留英文动词/名词等实词，虚词和冠词隐藏为下划线填空，提示用户主动补全。
   - **无字幕**：完全隐藏所有字幕，用于纯听力训练和跟读。
   - **听写填空字幕**：每句英文随机隐藏1-2个关键词，用户可点击输入答案后查看正确文本，用于精确听辨训练。
   
   > **说明**：以上所有模式的数据来源均为中英双语字幕。若影片仅有单语字幕，则中英双语模式降级为纯英模式，关键词提示和听写填空仍基于英文字幕运行。
4. **逐句播放**：播放器自动识别字幕中的句子边界，用户可按"上一句"/"下一句"逐句跳转和循环。
5. **遍数计数**：当一段内容（AB区间内）完整播放一次，自动计入"有效遍数"。有效遍数判定规则：播放完成度>90%且播放时长>=片段理论时长×0.9。
6. **当前遍数显示**：播放器右上角常驻显示"第 X 遍 / 100遍"。
7. **快捷操作（桌面端 - 键盘）**：
   - 空格键：播放/暂停
   - 左右方向键：快进/快退 5 秒
   - 上/下方向键：上一句/下一句
   - A键：标记/清除 A 点
   - B键：标记/清除 B 点
   - S键：循环切换字幕模式（按中英→纯英→关键词→无字幕→听写→中英的顺序）

8. **触控操作（移动端 - 手势）**：
   - **单击画面**：播放/暂停
   - **左右滑动**：快进/快退 5 秒（滑动距离>屏幕宽度的10%触发）
   - **长按画面（>1秒）**：设置/清除 AB 循环点（第一次长按=A点，第二次长按=B点，第三次长按=清除AB区间）
   - **双击画面**：切换字幕模式（按中英→纯英→关键词→无字幕→听写→中英的顺序循环）
   - **双指捏合**：缩放字幕字号（放大/缩小）
   - **从屏幕底部边缘上滑**：呼出控制面板（变速选择、字幕模式切换、阶段信息、当前遍数详情）
   - **左右滑动字幕区域**：上一句/下一句（仅限播放器底部字幕条区域）

**交互流程**：
1. 用户从首页点击视频项目，进入播放器。
2. 首次进入时，播放器默认加载第一个AB区间（若用户未设置，则默认为视频前5分钟或前10句字幕）。
3. **桌面端**：用户按A键标记当前时间点为A，按B键标记为B，AB区间自动开始循环播放。
   **移动端**：用户长按画面（>1秒），第一次长按标记A点，第二次长按标记B点，AB区间自动开始循环播放。长按超过1秒后触觉反馈（震动）确认标记成功。
4. **桌面端**：用户按S键切换字幕模式，字幕层即时刷新。
   **移动端**：用户双击画面切换字幕模式，或从底部上滑呼出控制面板后点击字幕模式按钮切换。
5. 每完成一次有效播放，右上角遍数+1，并显示短暂浮层"第 X 遍完成"。
6. 当遍数达到当前阶段上限（如第1阶段为10遍），播放器暂停并弹窗提示"恭喜完成第1阶段！是否进入第2阶段？"。

**异常处理**：

| 异常场景 | 处理方式 | 用户提示 |
|---------|---------|---------|
| AB区间过短（<1秒） | 提示并自动延长至1秒 | "AB区间太短，已自动调整为1秒" |
| 变速后音频失真 | 检测到失真时自动回退至1.0x | "当前变速音频质量不佳，已恢复常速" |
| 字幕模式切换时无对应字幕数据 | 降级显示可用字幕（如听写模式需要分析词性，若无法分析则降级为纯英） | "当前影片暂不支持听写模式，已切换为纯英文字幕" |
| 播放过程中视频文件被移动或删除 | 检测到文件缺失时暂停播放，提示用户重新定位文件 | "视频文件找不到啦，点击重新选择文件位置" |
| **移动端：后台播放被系统中断** | 保存当前播放进度和遍数，用户切回前台时自动恢复 | "学习已暂停，欢迎回来继续" |
| **移动端：来电或系统通知打断播放** | 暂停播放，通话结束后自动恢复（若用户未手动退出App） | —（后台自动处理） |
| **移动端：横竖屏切换** | 播放器自适应切换布局，横屏时全屏播放，竖屏时视频占上半屏+控制区占下半屏 | —（自适应布局，无弹窗） |
| 连续播放超过2小时 | 弹出休息提醒，建议用户休息5分钟 | "你已经连续学习2小时了，休息一下眼睛吧" |

### 3.3 阶段化学习引擎

**功能描述**：将100遍自动划分为10个阶段，每个阶段解锁不同的字幕模式、学习任务和学习提示，让100遍不再单调。

**用户故事**：
- 作为自学者，我希望知道每10遍该做什么、看什么字幕，以便不陷入机械重复。
- 作为自学者，我希望每完成一个阶段获得成就感，以便有动力继续下一个10遍。

**优先级**：P0

**业务规则**：
1. 100遍划分为10个阶段，每阶段10遍。阶段划分规则如下：

| 阶段 | 遍数范围 | 默认字幕模式 | 核心任务 | 解锁提示 |
|------|---------|-------------|---------|---------|
| 第1阶段 | 1-10 | 中英双语 | 理解剧情，熟悉整体语境 | "先理解大意，不用纠结每个词" |
| 第2阶段 | 11-20 | 纯英文字幕 | 训练英文阅读能力，注意句型结构 | "尝试不依赖中文，猜不懂的词" |
| 第3阶段 | 21-30 | 关键词提示 | 训练听力抓关键词能力 | "只保留核心词，逼自己听细节" |
| 第4阶段 | 31-40 | 无字幕 | 纯听力训练，建立语音直觉 | "闭上眼睛，感受语调和节奏" |
| 第5阶段 | 41-50 | 听写填空 | 训练精确听辨能力 | "写下你听到的每个词" |
| 第6阶段 | 51-60 | 无字幕 | 影子跟读：比原声慢0.5秒跟读 | "像影子一样跟着说" |
| 第7阶段 | 61-70 | 无字幕 | 同步跟读：与原声同步朗读 | "试着和演员同时开口" |
| 第8阶段 | 71-80 | 无字幕 | 变速挑战：1.25x 播放跟读 | "加快速度，训练反应力" |
| 第9阶段 | 81-90 | 无字幕 | 独立朗读：视频静音，自己配音 | "现在由你来演这段戏" |
| 第10阶段 | 91-100 | 无字幕 | 完美演绎：原速流畅跟读，力争与原声一致 | "最后一次，完美收尾" |

2. 每阶段完成后，系统自动弹窗祝贺并展示阶段徽章，询问是否进入下一阶段。
3. 用户可手动调整阶段配置：修改每阶段的遍数（如从10遍改为5遍或20遍）、修改默认字幕模式、修改阶段提示文案。修改仅对当前影片生效。
4. 用户可跳过当前阶段（标记为"已完成"），但建议不跳过超过3个阶段。
5. 阶段进度在播放器中以进度条+阶段节点的方式常驻显示。
6. 若用户在某阶段中断超过7天，重新进入时弹窗提示"欢迎回来！建议从上一阶段最后2遍开始复习"。

**交互流程**：
1. 用户首次进入某影片的播放器，阶段化引擎自动加载默认阶段配置。
2. 播放器顶部显示阶段进度条：10个节点，当前节点高亮，已完成节点打勾。
3. 每完成一阶段的最后一遍，播放器暂停，屏幕中央弹出阶段完成弹窗：
   - 标题："阶段 X 完成！"
   - 内容：累计学习时长、该阶段平均语速、鼓励语
   - 操作按钮："休息一下" / "进入下一阶段"
4. 用户点击"进入下一阶段"，字幕模式自动切换为下一阶段的默认模式，遍数归零重计。
5. 用户可在播放器设置中点击"阶段配置"，进入阶段编辑页修改规则。

**异常处理**：

| 异常场景 | 处理方式 | 用户提示 |
|---------|---------|---------|
| 用户频繁跳过阶段（连续跳过3个） | 弹出温和提醒，建议至少完成部分遍数 | "跳过快了可能会影响效果哦，建议在无字幕阶段至少练5遍" |
| 影片时长过短（<5分钟） | 阶段划分仍然生效，但提示用户100遍可能过快完成 | "这部影片较短，100遍大约需要X小时，建议换长一点的影片" |
| 阶段配置被用户修改为空 | 拒绝保存，恢复默认配置 | "阶段配置不能为空，已恢复默认设置" |
| 某阶段要求的字幕模式当前不可用 | 自动降级为可用的最接近模式（如听写不可用时降级为纯英） | "听写模式暂不可用，已切换为纯英文字幕" |

### 3.4 进度追踪

**功能描述**：全面记录用户的学习进度，包括100遍计数、学习时长、阶段完成情况，并以可视化方式呈现。

**用户故事**：
- 作为自学者，我希望看到自己在第几遍了，以便获得继续推进的成就感。
- 作为自学者，我希望看到累计学习时长，以便了解自己的投入程度。

**优先级**：P0

**业务规则**：
1. **影片维度进度**：每部影片独立记录：当前阶段、当前阶段已完成的遍数、总完成遍数（上限100）、累计学习时长、最后学习时间。
2. **全局维度进度**：所有影片的汇总数据：累计学习天数、累计学习时长、已完成影片数、当前连胜天数。
3. **100遍计数规则**：仅在当前AB区间内完成有效播放时计数。用户主动暂停不计数。切换AB区间后，新区间的遍数独立计数（即一部影片可以有多个AB区间，每个区间各自向100遍推进）。
4. **可视化组件**：
   - 影片卡片：展示影片封面、影片名、当前阶段（如"阶段 3/10"）、总遍数进度条（0-100%）。
   - 详情仪表盘：日历热力图（绿色深浅表示当日学习时长）、阶段节点图、累计时长数字。
   - 首页总览：当前连胜天数、今日学习时长、最近学习影片快捷入口。
5. **数据持久化**：所有进度数据保存在本地数据库/文件中，不上传云端。
6. **数据导出/导入**：支持将学习记录导出为JSON文件（含影片元数据、进度、阶段配置、生词本），方便用户自行备份或在桌面端与移动端之间手动迁移学习记录。
   - **导出**：用户可选择导出单部影片或全部影片的数据，生成JSON文件保存到系统指定位置。
   - **导入**：用户选择之前导出的JSON文件，App合并导入数据（同名影片以最新学习时间为准覆盖，或提示用户选择保留/覆盖）。
   - **跨设备迁移场景**：用户在桌面端导出JSON → 通过微信/邮件/AirDrop发送到手机 → 手机端导入JSON，继续学习。

**交互流程**：
1. 用户打开App，首页顶部展示全局概览：连胜天数、今日时长、总影片数。
2. 首页中部为影片卡片列表，按"最近学习"排序。
3. 用户点击某影片卡片的"查看详情"，进入该影片的进度仪表盘。
4. 仪表盘默认展示"阶段节点图"，用户可切换至"日历热力图"或"学习统计"。
5. 在仪表盘设置中，用户可点击"导出数据"，选择CSV或JSON格式，保存到本地指定路径。

**异常处理**：

| 异常场景 | 处理方式 | 用户提示 |
|---------|---------|---------|
| 本地数据文件损坏 | 尝试从自动备份恢复（保留最近3个备份），失败则提示重置 | "数据文件异常，已尝试自动恢复。如仍有问题，可能需要重置进度" |
| 用户手动修改数据文件 | 检测到数据校验失败时拒绝加载，提示从备份恢复 | "数据校验失败，建议从自动备份恢复" |
| 累计时长计算异常（如系统时间回退） | 以播放器的实际播放时间戳累加为准，不依赖系统时间差 | —（后台自动处理，用户无感知） |

### 3.5 习惯养成系统

**功能描述**：通过连胜机制、每日提醒和基础徽章，帮助用户建立持续学习的习惯。

**用户故事**：
- 作为自学者，我希望看到连续学习了多少天，以便不愿意因为断签而破坏连胜记录。
- 作为自学者，我希望在忘记学习时收到提醒，以便保持学习节奏。

**优先级**：P1

**业务规则**：
1. **连胜机制**：用户每日完成至少1次有效学习（任意影片至少1遍），当日计入连胜。连续7天/30天/100天分别解锁不同等级徽章。
2. **断签处理**：断签后连胜归零，不可补签（自用工具，简化设计）。断签超过3天后，首次学习时弹窗鼓励"欢迎回来，从头开始也是进步"。
3. **每日提醒**：用户可设置每日提醒时间（如21:00）。到达提醒时间时，若当日尚未学习，系统推送本地通知"今天的100LS练了吗？"。
4. **徽章系统（MVP简化版）**：
   - 初出茅庐：完成第1遍
   - 持之以恒：连续7天学习
   - 百遍勇士：完成任意影片100遍
   - 多面手：同时推进3部影片
   - 早起鸟：在上午8点前完成学习
5. 提醒功能需用户授权系统通知权限，未授权时App内以红点提示代替推送。

**交互流程**：
1. 用户首次进入App时，弹出"设置学习提醒"引导页，可设置每日提醒时间或选择"暂不设置"。
2. 用户完成每日首次有效学习后，首页连胜天数+1，并显示"连胜X天"徽章。
3. 若用户连续学习达到7天，全屏弹窗展示"持之以恒"徽章动画。
4. 到达提醒时间，系统发送本地通知；用户点击通知直接进入App首页。
5. 用户可在"我的"-"习惯设置"中修改提醒时间或关闭提醒。

**异常处理**：

| 异常场景 | 处理方式 | 用户提示 |
|---------|---------|---------|
| 用户关闭系统通知权限 | 降级为App内红点提醒，不强制要求开启 | —（设置页显示"通知已关闭，将以App内提醒代替"） |
| 用户修改系统日期试图刷连胜 | 以播放器实际学习的时间戳记录，日期异常时不计入连胜 | —（后台自动防作弊，用户无感知） |
| 提醒时间到达时App正在播放视频 | 不弹窗打断播放，改为播放完成后底部轻提示"今日已完成，连胜+1" | — |

### 3.6 词汇辅助

**功能描述**：在学习过程中支持点击字幕查词、标记生词，降低因生词卡壳导致的挫败感。

**用户故事**：
- 作为自学者，我希望看到不认识的词能立即查看释义，以便不被生词打断学习节奏。
- 作为自学者，我希望标记出需要复习的生词，以便后续集中复习。

**优先级**：P1

**业务规则**：
1. **点击查词**：播放器中，用户点击字幕中的任意单词，弹出浮层展示该词的释义（MVP阶段接入本地词典或开源词典数据，暂不提供整句翻译）。
2. **生词标记**：用户可将单词标记为"生词"，被标记的单词在后续播放中以高亮底色显示。
3. **生词本**：独立页面展示所有标记过的生词列表，按影片分类，支持排序（按标记时间/按字母）。
4. **释义来源**：MVP阶段使用开源英汉词典数据（如CC-CEDICT），释义包含音标、词性、中文释义，不含例句和发音。
5. **跨语言支持**：若影片为非英语（如日语），查词同样支持对应语言的词典查询（依赖对应语言的开源词典数据）。

**交互流程**：
1. 播放器中：
   - **桌面端**：用户鼠标悬停字幕中的单词，该单词高亮；点击单词弹出查词浮层。
   - **移动端**：用户长按字幕中的单词（>0.5秒），该单词高亮并弹出查词浮层。
2. 查词浮层展示：单词、音标、词性、中文释义、"标记为生词"按钮。
3. 用户点击"标记为生词"，按钮变为"已标记"，该单词在后续播放中自动高亮。
4. 用户从播放器侧边栏（桌面端）或底部Tab栏（移动端）的"生词本"入口进入生词列表页。
5. 生词本支持搜索、按影片筛选、取消标记。

**异常处理**：

| 异常场景 | 处理方式 | 用户提示 |
|---------|---------|---------|
| 点击的单词不在词典中 | 展示"暂无释义"，允许用户手动添加释义 | "该词暂未收录，你可以手动添加释义" |
| 字幕文本是中文（中英双语模式下） | 点击中文词不触发查词，仅英文词可点 | —（仅英文词响应点击） |
| 词典数据文件缺失或损坏 | 提示用户重新下载词典包 | "词典数据异常，请重新下载" |

## 4. 非功能需求

| 类别 | 要求 | 验收标准 |
|------|------|---------|
| 性能 | 播放器启动到可播放时间 | < 2秒（本地视频） |
| 性能 | 字幕切换响应时间 | < 300ms |
| 性能 | AB循环切换无感知延迟 | 循环点切换时无卡顿、无黑屏 |
| 性能 | 自动字幕生成速度 | 处理时长 <= 视频时长的 1.5 倍（如10分钟视频在15分钟内完成） |
| 兼容性 | 支持平台 | macOS 13+ / Windows 10+ / **iOS 16+ / Android 10+** |
| 兼容性 | 视频解码 | 支持H.264/H.265/AV1硬件解码（桌面端）；移动端优先H.264硬件解码 |
| 兼容性 | 字幕渲染 | 支持UTF-8编码，支持特殊字符和emoji |
| 兼容性 | **移动端屏幕适配** | **支持横竖屏切换，竖屏时播放器采用"视频上半屏+控制下半屏"布局，横屏时全屏播放** |
| 兼容性 | **移动端文件系统** | **支持iOS Files App、Android系统文件管理器、相册分享导入** |
| 数据安全 | 用户数据存储 | 所有数据（进度、字幕、设置）仅保存在本地，不上传任何服务器 |
| 数据安全 | 视频文件保护 | App不复制/转码用户视频，仅读取原始文件路径 |
| 数据安全 | 导出数据脱敏 | 导出的学习记录不含任何视频文件内容，仅包含进度元数据 |
| 可用性 | 离线使用 | 除首次下载词典数据和自动字幕生成外，所有核心功能完全离线可用 |
| 可用性 | 崩溃恢复 | 播放器异常退出后，重新打开可恢复至上一次的播放位置、AB区间和遍数 |

## 9. 后续版本迭代规划

> 以下功能在本期MVP中明确不做，但已纳入产品路线图，供后续版本参考。

### V2.0：视频自动切分

**背景**：URL下载的视频往往是一整部电影或一集剧集（时长20-120分钟），对于100LS学习者来说，一次性处理这么长的内容负担较重。学习者通常只需要反复练习其中几个经典场景（如5-10分钟的对话片段）。

**功能描述**：
- 系统通过音频分析（静音检测、语速变化、场景切换检测）自动识别视频中的场景边界。
- 将长视频自动切分为多个短视频片段（每个片段3-10分钟）。
- 每个片段独立进行100LS学习（独立的100遍计数、阶段推进、进度追踪）。
- 用户也可手动设置切分点，或在自动切分结果上微调。
- 切分后的片段保留原视频的双语字幕数据，时间轴自动重新计算。

**价值**：大幅降低单部影片的学习门槛，用户可以从一个5分钟片段开始，逐步扩展到更多片段，避免因"一整部电影太长"而望而却步。

**优先级**：V2.0 P0

### V2.0+ 候选功能
