## 一、需求全貌梳理

### 涉及角色

| 角色 | 说明 |
|------|------|
| 自学者（主角色） | 有一定英语基础，想用100LS方法学习，通过App完成视频下载、播放、阶段学习和进度追踪 |
| 多语言爱好者（次要） | 使用同一套流程学习非英语内容 |

### 核心流程

```
粘贴URL/本地导入 → 解析/下载 → 字幕获取/生成/翻译/对齐 → 预览确认 → 进入播放器
→ AB循环播放 → 遍数计数 → 阶段自动推进 → 进度追踪 → 习惯养成 → 词汇辅助
```

### 业务规则要点

1. 视频来源：URL下载（YouTube/Bilibili）+ 本地导入
2. 中英双语字幕获取优先级：平台双语字幕 → 单语字幕+翻译 → 语音识别+翻译
3. 100遍分为10阶段，每阶段10遍，默认字幕模式自动切换
4. 遍数计数规则：播放完成度>90%且时长>=理论时长×0.9
5. 数据纯本地存储，跨设备通过JSON导出/导入手动迁移
6. 桌面端快捷键 + 移动端触控手势两套交互体系

### 数据实体

| 实体 | 核心字段 |
|------|---------|
| 视频项目 | ID、名称、来源URL、本地路径、时长、语言、导入时间、状态 |
| 双语字幕 | 句子ID、英文原文、中文译文、开始时间、结束时间、影片ID |
| 学习进度 | 影片ID、当前阶段、当前阶段遍数、总遍数、累计时长、最后学习时间 |
| AB区间 | 影片ID、A点时间、B点时间、当前遍数 |
| 生词 | 单词、音标、词性、释义、影片ID、标记时间 |
| 连胜记录 | 当前连胜天数、最后学习日期、历史最高连胜 |

## 二、Epic 与 Story 拆解

### Epic 1：视频导入与字幕管理

> **拆分模式**：按工作流步骤（URL解析 → 下载 → 字幕获取 → 字幕合并 → 本地导入 → 编辑 → 预览）

---

#### US-001：粘贴视频URL并解析元数据

**角色**：作为自学者
**需求**：我希望粘贴一个视频链接后App自动解析出标题、时长、封面等信息
**价值**：以便确认这是我想学习的视频，再决定是否下载
**优先级**：P0
**Story Points**：2

**Acceptance Criteria**：
- [ ] Given 用户复制了一个有效的YouTube视频链接并粘贴到输入框，When 用户点击"解析"按钮，Then 系统在3秒内展示视频标题、时长、封面缩略图和平台来源
- [ ] Given 用户粘贴了一个无效的URL（如普通网页链接），When 用户点击"解析"，Then 系统提示"无法解析该链接，请检查URL是否正确"

**依赖**：无
**技术备注**：需集成各平台的元数据解析接口（不下载视频，仅获取信息）

---

#### US-002：下载视频文件到本地

**角色**：作为自学者
**需求**：我希望确认视频信息后App自动下载视频到本地
**价值**：以便离线观看和反复学习
**优先级**：P0
**Story Points**：3

**Acceptance Criteria**：
- [ ] Given 用户在解析结果页点击"开始下载"，When 下载进行时，Then 系统显示视频下载进度条（百分比+已下载大小/总大小）
- [ ] Given 下载过程中网络中断，When 网络恢复后，Then 系统自动断点续传，最多自动重试3次，失败则提示用户手动重试
- [ ] Given 视频下载完成，When 用户查看本地文件，Then 视频文件保存在App指定的下载目录中，格式为MP4

**依赖**：US-001
**技术备注**：下载目录需用户可配置；大文件需分段下载

---

#### US-003：从视频平台下载可用字幕

**角色**：作为自学者
**需求**：我希望下载视频时App能同时抓取平台提供的所有可用字幕
**价值**：以便获得最准确的字幕数据，减少后续语音识别和翻译的工作量
**优先级**：P0
**Story Points**：3

**Acceptance Criteria**：
- [ ] Given 某YouTube视频平台提供了英文字幕和中文字幕，When 用户下载该视频，Then 系统同时下载英字和中字两个字幕文件，保存到与视频同目录
- [ ] Given 某视频平台仅提供了英文字幕，When 用户下载该视频，Then 系统下载英文字幕，并标记"缺少中文字幕，将自动翻译生成"
- [ ] Given 某视频平台无任何字幕，When 用户下载该视频，Then 系统标记"无平台字幕，将使用语音识别生成"

**依赖**：US-002（可与US-002并行启动，但需等视频元数据解析完成）
**技术备注**：不同平台的字幕格式和API不同，需适配

---

#### US-004：通过语音识别生成英文字幕

**角色**：作为自学者
**需求**：我希望对于没有字幕的视频，App能自动识别语音并生成英文字幕
**价值**：以便即使没有现成字幕也能开始100LS学习
**优先级**：P0
**Story Points**：5

**Acceptance Criteria**：
- [ ] Given 某视频无可用字幕且已下载完成，When 用户触发"生成字幕"，Then 系统在视频时长1.5倍时间内完成语音识别，生成带时间轴的英文字幕文件
- [ ] Given 视频中有背景音乐干扰或多人同时说话，When 语音识别完成后，Then 系统生成字幕，并在时间轴编辑器中标记"识别置信度低"的句子（用黄色底色提示）
- [ ] Given 语音识别过程中用户切换到其他App（移动端），When 用户切回本App，Then 识别任务在后台继续运行，进度条实时更新

**依赖**：无（可与US-003并行）
**技术备注**：桌面端优先本地处理；移动端因性能限制，超过15分钟的视频建议在桌面端处理

---

#### US-005：英文字幕机器翻译为中文字幕

**角色**：作为自学者
**需求**：我希望仅有英文字幕的视频能自动生成对应的中文字幕
**价值**：以便在渐进式学习初期能通过中英对照理解剧情
**优先级**：P0
**Story Points**：3

**Acceptance Criteria**：
- [ ] Given 某影片仅有英文字幕（无中文字幕），When 系统完成字幕翻译，Then 每个英文句子都有对应的中文翻译，译文显示在预览页
- [ ] Given 英文句子包含俚语或文化特定表达（如"break a leg"），When 翻译完成后，Then 系统尽可能提供意译（如"祝你好运"）而非直译
- [ ] Given 翻译过程中网络中断，When 网络恢复后，Then 系统自动恢复翻译，已翻译的句子不重复处理

**依赖**：US-004（或US-003，当平台仅有英文字幕时）
**技术备注**：使用机器翻译API或本地翻译模型；需处理API限流和失败回退

---

#### US-006：中英字幕按时间轴对齐合并为双语字幕

**角色**：作为自学者
**需求**：我希望中英两个字幕文件能自动按时间轴对齐合并成一条双语字幕
**价值**：以便播放器能在不同阶段切换中英对照、纯英、无字幕等模式
**优先级**：P0
**Story Points**：5

**Acceptance Criteria**：
- [ ] Given 系统已获取英文字幕和中文字幕（各有100句），When 执行对齐合并，Then 生成的双语字幕有100个条目，每条包含英文原文、中文译文、开始时间、结束时间，且中文译文与对应英文句子的时间戳偏差<500ms
- [ ] Given 英文字幕有100句但中文字幕仅有80句（中文翻译合并了部分句子），When 执行对齐合并，Then 系统以英文句子为基准拆分，缺失对应中文的句子标记为"[待补充]"，并在时间轴编辑器中高亮提示
- [ ] Given 对齐后中英文句数差异超过30%，When 合并完成后，Then 影片状态标记为"字幕对齐异常"，预览页显示警告图标

**依赖**：US-003 和 US-005（需中英双语字幕数据都就绪）
**技术备注**：对齐算法以时间戳最近为原则，需处理字幕文件格式差异（SRT/VTT/ASS）

---

#### US-007：本地视频文件导入

**角色**：作为自学者
**需求**：我希望从本地文件系统导入已有的视频和字幕文件
**价值**：以便使用已经下载好的影片资源
**优先级**：P0
**Story Points**：2

**Acceptance Criteria**：
- [ ] Given 用户电脑某目录下有 video.mp4 和 video_en.srt / video_cn.srt，When 用户选择 video.mp4 导入，Then App自动检测同目录中英字幕文件，加载并进入预览页
- [ ] Given 用户选择了一个 .wmv 文件，When 用户确认导入，Then App提示"该格式暂不支持，请转换为MP4后重试"
- [ ] Given 用户从iOS相册通过"分享到App"导入了一个MP4视频，When 导入完成后，Then 视频项目出现在首页列表，状态为"未开始"

**依赖**：无（与URL下载路径独立）
**技术备注**：移动端需处理系统文件权限和相册访问

---

#### US-008：字幕时间轴编辑器

**角色**：作为自学者
**需求**：我希望在预览页能调整字幕的时间轴和文本内容
**价值**：以便修正自动对齐或语音识别产生的偏差
**优先级**：P0
**Story Points**：3

**Acceptance Criteria**：
- [ ] Given 用户进入字幕时间轴编辑器，When 用户将某句字幕的开始时间+300ms，Then 播放器预览中该句字幕的显示时间同步延后300ms
- [ ] Given 中英字幕对齐后某句中文翻译不正确，When 用户点击该句中文文本并修改，Then 修改后的文本即时生效，并在播放器预览中更新
- [ ] Given 字幕整体时间轴偏移（如比画面快2秒），When 用户在"整体偏移"滑块上-2000ms，Then 所有字幕的时间戳统一偏移-2000ms

**依赖**：US-006 或 US-007
**技术备注**：编辑器需支持逐句粒度调整+整体偏移两种模式

---

#### US-009：预览确认页

**角色**：作为自学者
**需求**：我希望在正式学习前预览视频前30秒的字幕效果
**价值**：以便确认字幕质量和同步效果，不满意可以返回调整
**优先级**：P0
**Story Points**：2

**Acceptance Criteria**：
- [ ] Given 视频和字幕处理完成，When 用户进入预览确认页，Then 页面展示视频前30秒的画面，字幕按当前默认模式（中英双语）叠加显示
- [ ] Given 用户在预览页发现字幕不同步，When 用户点击"调整字幕"，Then 跳转至字幕时间轴编辑器
- [ ] Given 用户对预览效果满意，When 用户点击"确认"，Then 视频项目出现在首页列表，状态为"未开始"，页面返回首页

**依赖**：US-006/US-007 和 US-008
**技术备注**：预览页复用播放器核心组件，但限制播放范围为前30秒

### Epic 2：100LS专用播放器

> **拆分模式**：按工作流步骤（播放控制 → AB循环 → 变速 → 字幕切换 → 逐句 → 遍数计数 → 交互适配）

---

#### US-010：视频播放器基础功能

**角色**：作为自学者
**需求**：我希望播放器支持基本的播放/暂停、进度拖拽和音量调节
**价值**：以便控制学习节奏
**优先级**：P0
**Story Points**：3

**Acceptance Criteria**：
- [ ] Given 用户进入某影片的播放器，When 页面加载完成，Then 播放器在2秒内开始播放视频，画面无黑屏或卡顿
- [ ] Given 视频正在播放，When 用户点击播放/暂停按钮（或按空格键/单击屏幕），Then 视频立即暂停/恢复播放，响应延迟<100ms
- [ ] Given 用户拖拽进度条，When 释放拖拽，Then 视频跳转到对应时间点，字幕同步更新

**依赖**：US-009（视频项目已就绪）
**技术备注**：播放器需支持H.264/H.265硬件解码；移动端优先H.264

---

#### US-011：AB循环功能

**角色**：作为自学者
**需求**：我希望设置起点A和终点B，让播放器在区间内自动循环播放
**价值**：以便反复观看同一段台词，加深记忆
**优先级**：P0
**Story Points**：5

**Acceptance Criteria**：
- [ ] Given 用户在桌面端播放器中按A键，When 当前播放时间为00:01:00，Then 系统记录A点为00:01:00，屏幕边缘显示A标记
- [ ] Given 用户在移动端长按画面1秒，When 系统震动反馈后，Then 记录A点（第二次长按记录B点，第三次长按清除AB区间）
- [ ] Given A点和B点都已设置（A=00:01:00，B=00:01:10），When 视频播放到B点，Then 视频自动跳回A点继续播放，形成无限循环
- [ ] Given 用户设置的AB区间只有0.5秒，When 确认设置时，Then 系统自动延长至1秒，并提示"AB区间太短，已自动调整为1秒"

**依赖**：US-010
**技术备注**：AB区间数据需持久化，崩溃后可恢复

---

#### US-012：变速播放

**角色**：作为自学者
**需求**：我希望调整视频播放速度（0.5x到1.5x），且变速后声音不失真
**价值**：以便跟不上语速时降低速度，熟练后加速挑战
**优先级**：P0
**Story Points**：3

**Acceptance Criteria**：
- [ ] Given 当前播放速度为1.0x，When 用户在控制面板选择0.75x，Then 播放速度降低，语音音调保持不变，无明显失真
- [ ] Given 用户在移动端呼出底部控制面板，When 用户点击变速按钮，Then 弹出0.5x/0.75x/1.0x/1.25x/1.5x五档选择，当前档位高亮
- [ ] Given 当前变速后音频质量不佳（某些编码格式不支持变速不变调），When 系统检测到失真，Then 自动回退至1.0x，并提示"当前变速音频质量不佳，已恢复常速"

**依赖**：US-010
**技术备注**：变速不变调依赖音频处理算法，不同平台实现可能不同

---

#### US-013：中英双语字幕渲染与切换

**角色**：作为自学者
**需求**：我希望播放器能在5种字幕模式之间切换（中英双语/纯英/关键词提示/无字幕/听写填空）
**价值**：以便按阶段逐步从依赖字幕过渡到盲听
**优先级**：P0
**Story Points**：5

**Acceptance Criteria**：
- [ ] Given 当前字幕模式为"中英双语"，When 用户在桌面端按S键（或移动端双击画面/呼出控制面板切换），Then 字幕模式切换为"纯英文字幕"，画面中央短暂显示"纯英文字幕"Toast提示，切换延迟<300ms
- [ ] Given 当前字幕模式为"关键词提示"，When 视频显示字幕"I have been waiting for you"，Then 虚词被隐藏为下划线，显示为"I ___ been waiting ___ you"
- [ ] Given 当前字幕模式为"听写填空"，When 视频显示字幕"I love learning languages"，Then 系统随机隐藏1-2个关键词（如"love"和"languages"），用户点击空白处可输入答案
- [ ] Given 某影片仅有单语字幕（无中英双语数据），When 用户尝试切换到"中英双语"模式，Then 系统自动降级为"纯英文字幕"模式，并提示"当前影片暂不支持双语模式"

**依赖**：US-010 和 US-006（需双语字幕数据）
**技术备注**：字幕渲染层需独立于视频层，支持CSS/Canvas动态切换；关键词提示需词性分析

---

#### US-014：逐句播放功能

**角色**：作为自学者
**需求**：我希望播放器能按字幕句子边界逐句跳转和循环
**价值**：以便聚焦练习某一句台词的发音和语调
**优先级**：P0
**Story Points**：3

**Acceptance Criteria**：
- [ ] Given 当前播放到第5句字幕，When 用户在桌面端按下方向键（或移动端左右滑动字幕区域），Then 播放器跳转到第6句的开头并暂停，等待用户播放
- [ ] Given 用户开启了逐句循环模式，When 当前句子播放完毕，Then 播放器自动跳回该句开头循环播放，直到用户手动切换到下一句
- [ ] Given 视频无字幕数据，When 用户尝试使用逐句播放，Then 按钮置灰不可用，提示"当前影片无字幕，无法使用逐句播放"

**依赖**：US-010 和 US-013（需字幕时间轴数据）
**技术备注**：句子边界基于字幕时间轴的断句，不依赖音频分析

---

#### US-015：遍数计数与有效播放判定

**角色**：作为自学者
**需求**：我希望播放器能自动统计我在AB区间内完成了多少遍有效播放
**价值**：以便清晰掌握学习进度，获得成就感
**优先级**：P0
**Story Points**：3

**Acceptance Criteria**：
- [ ] Given 当前AB区间为00:01:00-00:01:10，用户已连续完整播放8遍，When 用户完成第9遍播放（播放完成度>90%且时长>=9秒），Then 右上角遍数从8变为9，画面中央短暂浮层显示"第9遍完成"
- [ ] Given 用户在第5遍播放到一半时点击暂停，When 用户恢复播放并完成剩余部分，Then 该次播放仍计入有效遍数（以总完成度判定）
- [ ] Given 用户在AB区间内快进跳过了中间3秒，When 播放到达B点时，Then 系统判定本次播放完成度<90%，不计入有效遍数，无"第X遍完成"提示

**依赖**：US-011

## 五、关键决策与风险提示

### 可并行开发的Story组

| 组号 | Story | 说明 |
|-----|-------|------|
| A组 | US-001, US-004, US-007 | URL解析、语音识别、本地导入互不依赖，可并行 |
| B组 | US-010, US-018, US-028 | 播放器基础、阶段配置、每日提醒互不依赖，可并行 |
| C组 | US-022, US-030 | 进度记录和查词可并行开发 |

### 关键路径（不可压缩）

```
US-001 → US-002 → US-003 → US-006 → US-009 → US-010 → US-011 → US-015 → US-019
```

这条路径决定了MVP的最早上线时间。即使所有其他Story都完成了，缺少这条路径上的任何一个环节，用户都无法完成一次完整的100LS学习闭环。

### Story Point > 8 的风险检查

本次拆解中所有Story Points均 ≤ 5，无需进一步拆分。最大的Story是 US-004（语音识别，5 pts）、US-006（双语对齐，5 pts）、US-011（AB循环，5 pts）、US-013（字幕切换，5 pts）、US-017（移动端手势，5 pts）、US-024（详情仪表盘，5 pts）。这些Story在1个Sprint内可完成，粒度合适。
